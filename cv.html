0 → 1 foundational infrastructure giving advertisers direct control over how their product data appears in ads, for every objective.

Advertisers upload their products to Meta's catalog ↗, a database of everything they sell including products with images, titles and prices. That data powers every ad and storefront on Facebook and Instagram. Sophisticated advertisers use multiple sources to keep it current: a platform like Shopify for live prices and stock, and a data feed for ad-optimised titles and images. When two sources set a different value for the same attribute, Meta has to pick one. Before this project, advertisers had no say in that decision.

The problem showed up differently depending on who you talked to. One group maintained a single catalog but couldn't control which source won for each attribute, so their products showed up wrong across different placements. Another group worked around the lack of control by duplicating their entire catalog: one for their Shop, one for ads, one per region. Each new catalog allowed for creative control, but came at the cost of ads performance.

This was a two-part solution. Configure Sources gave advertisers control over which data source set each product attribute. Product Data Versions let them create alternate representations of their products for different marketing objectives.

This project is under NDA. Screens shown here have shipped publicly. Full case study is available on request.

Phase 1: Configure Sources

I co-led research and project scoping, and designed a solution to manage data sources with a transparency-first approach: showing advertisers which data source was winning for each attribute before asking them to make any decisions about it. Highlights shown below:

See what's winning

Attributes updated by multiple data sources are surfaced first. Advertisers can see which source is setting each value, like price, image, title, before making any changes.

Then change it

Advertisers reprioritise by dragging sources into order. Meta's default is visible and overridable, but at least one source must always be set so buyers never see wrong information that looses their trust.

Phase 2: Product Data Versions

I led the project from Alpha to Beta launch. Notably, I co-authored a technical behaviour proposal with engineering to establish the right data model before any interface work began, and built a design vision that became the roadmap for 5 product teams. In addition to designing, I was deeply involved in advertiser recruitment, research, and go-to-market preparation.

Outcomes

Configure Sources exceeded its resolution goal and opened new revenue opportunities for advertisers who previously couldn't use catalog-powered ads. Product Data Versions launched to Beta, and earned a GA recommendation with strong advertiser satisfaction and significant ROAS improvements1 reported across exit interviews.
